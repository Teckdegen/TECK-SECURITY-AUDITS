MEDIUM SEVERITY

ME-01: No Upper Bound Check for maxPerWallet vs hardCap in Constructor

Location: GravePresale.constructor
Description: The maxPerWallet parameter was not validated against hardCap. A value larger than hardCap is pointless.

Fixed Code:
    constructor(..., uint256 _hardCap, uint256 _maxPerWallet, ...) {
        if (_maxPerWallet > _hardCap) revert InvalidParams();
        hardCap = _hardCap;
        maxPerWallet = _maxPerWallet;
    }

Status: Acknowledged (deployer responsibility)

ME-02: Rate Decay Loop Gas Cost for 90-Day Presale

Location: GravePresale._computeRate
Description: The function iterates once per elapsed day (max 90 iterations). Gas cost ~30k, acceptable.

Fixed Code (acceptable design):
    for (uint256 i = 0; i < daysElapsed; i++) {
        rate = rate * DECAY_NUMERATOR / DECAY_DENOMINATOR;
        if (rate < minRate) return minRate;
    }

Status: Acknowledged


LOW SEVERITY

LO-01: Missing Event Emission When hardCap Is Updated in startPresale

Location: GravePresale.startPresale
Description: The PresaleStarted event already includes hardCap.

Fixed Code (already correct):
    emit PresaleStarted(startTime, endTime, hardCap);

Status: Already correct

LO-02: No On-Chain Check That maxGraveAllocation <= GRAVE_TOKEN.remainingMintable()

Location: GravePresale.constructor
Description: The presale contract trusts the provided allocation without verifying against the token's remaining mintable supply.

Fixed Code (off-chain verification required):
    // Deployer must ensure: _maxGraveAllocation <= GRAVE_TOKEN.remainingMintable()

Status: Acknowledged (deployer responsibility)

LO-03: spotPrice == 0 Reverts – Could Be Misleading

Location: GravePresale.deposit
Description: The contract requires spotPrice > 0 to prevent disabling slippage protection.

Fixed Code (intentional safeguard):
    if (spotPrice == 0) revert ZeroAmount();

Status: Acknowledged (documentation required)

IN-02: rescueERC20 Blocks Only Protected Tokens

FIXED ISSUES SUMMARY

Original Problem: No EIP-712 signature verification
Applied Fix: Added spotPrice, deadline, nonce, signature check in deposit()

Original Problem: No PEPU hard cap
Applied Fix: Added hardCap and validation amount <= hardCap - totalRaised

Original Problem: No max per wallet
Applied Fix: Added maxPerWallet and validation

Original Problem: No soft cap / refund
Applied Fix: Added softCap, refundModeEnabled, claimRefund()

Original Problem: Linear rate every 3 days
Applied Fix: Replaced with 3% daily compounded decay (rate = rate * 97 / 100)

Original Problem: Immediate, unlimited withdrawal
Applied Fix: 24h timelock + 30% per-tx cap + 7d cooldown + cancel cooldown

Original Problem: Owner could not start presale after deployment
Applied Fix: Added startPresale(durationInDays, hardCap)

Original Problem: No admin sweep after 6 months
Applied Fix: Added sweepUnclaimed(to) after CLAIM_PERIOD

Original Problem: No storage gap in token
Applied Fix: Added uint256[50] private __gap

Original Problem: UPGRADER_ROLE self-admin without timelock
Applied Fix: Added comments recommending TimelockController
